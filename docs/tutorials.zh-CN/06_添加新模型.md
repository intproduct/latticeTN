# 06 — 添加新模型

如何把 latticeTN 扩展到**新** Hamiltonian 模型（如 XXZ，或你自己的），同时
保持 AD 主线不变。现有的第二个模型 **TFI**（横场 Ising）就是模板。本教程
走完整流程：dense 参考 → MPO 构造器 → MPO-to-dense 测试 → native 能量测试
→ AD 求解器 → benchmark。

## 目标

- 端到端加一个新模型，不动 AD 主线、不改物理 convention。
- 保持 `S = σ/2`、`J`-scaling、开边界、`complex128`。
- 在新 MPO 上复用既有 `train_ad_*` 求解器。
- 加正确的测试，让新模型像 Heisenberg/TFI 一样被验证。

## AD 主线不变

加模型意味着加一个 **Hamiltonian**（dense 参考 + MPO 构造器）。求解器路径
不变：

```
MPS 参数 -> 可微 Rayleigh 商 -> loss.backward() -> optimizer.step()
```

新 MPO 只是插进 `rayleigh_energy_native(mps, mpo)` 和 `train_ad_*` 驱动。
**不**新增算法；loss 里**不**出现 SVD/QR/`eigh`；DMRG/Lanczos 仍只是参考
基线。

## 设计动机（写给中文用户）

为什么“加模型”而不是“加算法”？因为 latticeTN 的核心赌注是：**只要
Hamiltonian 能写成 MPO，可微 Rayleigh 商 + autograd 就能直接优化**。所以
扩展点是“提供新的 Hamiltonian 表达”，而不是“写新的求解器”。这也意味着新
模型必须同时给 **dense 参考**（小 N 金标准）和 **MPO**（可扩展可微形式），
并用 MPO-to-dense 测试证明两者一致——否则你无法相信 MPO 路径的数值。

## 流程（8 步）

### 1. dense 参考 — `latticetn/operators.py`

加 `def <model>_dense(N, ..., dtype=tc.complex128, device="cpu") -> tc.Tensor`，
返回 `(2**N, 2**N)` 的 dense Hamiltonian，用 `spin_operators(dtype, device)`
构建（**保持 `S = σ/2`**）。模板：`heisenberg_dense`、`tfi_dense`。

```python
# latticetn/operators.py
def xxz_dense(N, Jz=1.0, Jxy=1.0, dtype=tc.complex128, device="cpu") -> tc.Tensor:
    ops = spin_operators(dtype=dtype, device=device)   # S = sigma/2
    Sz, Sp, Sm = ops["Sz"], ops["S+"], ops["S-"]
    H = tc.zeros((2**N, 2**N), dtype=dtype, device=device)
    for i in range(N - 1):                              # 开边界
        SzSz = tc.kron(tc.kron(tc.eye(2**i, dtype=dtype, device=device), Sz),
                       tc.kron(Sz, tc.eye(2**(N-i-2), dtype=dtype, device=device)))
        SpSm = tc.kron(tc.kron(tc.eye(2**i, dtype=dtype, device=device), Sp),
                       tc.kron(Sm, tc.eye(2**(N-i-2), dtype=dtype, device=device)))
        SmSp = tc.kron(tc.kron(tc.eye(2**i, dtype=dtype, device=device), Sm),
                       tc.kron(Sp, tc.eye(2**(N-i-2), dtype=dtype, device=device)))
        H = H + Jz*SzSz + 0.5*Jxy*(SpSm + SmSp)        # SxSx+SySy = (1/2)(S+S-+S-S+)
    return H
```

### 2. MPO 构造器 — `latticetn/mpo.py`

在 `MPO` 类上加 `generate_<model>(self, ...)` 方法，填充
`self.tensors[i]`（`i in range(self.length)`）并返回 `self`。模板：
`generate_heisenberg`（D=5）、`generate_tfi`（D=3）。MPO 是 Hamiltonian 的
**MPS 形**分解；`to_dense()` 把它收缩回 dense 矩阵。

虚拟键的具体构造随模型而异：Heisenberg 构造器用 5 态键
（idle / 携带-Sz / 携带-S- / 携带-S+ / done），在每个键写 start/pair 算符。
近邻模型可推广此模式；更长程则增大键维 D。

### 3. MPO-to-dense 测试 — `tests/test_<model>_mpo_dense.py`

断言 MPO 收缩回 dense 参考，覆盖多个 N 与参数。模板：
`tests/test_tfi_mpo_dense.py`、`tests/test_heisenberg_mpo_dense.py`。

```python
# tests/test_xxz_mpo_dense.py
import torch as tc
from latticetn.mpo import MPO
from reference_models import xxz_dense, exact_ground_energy   # 从 operators 再导出

DTYPE = tc.complex128

def test_xxz_mpo_matches_dense_several_sizes():
    for N in [2, 3, 4, 5]:
        for (Jz, Jxy) in [(1.0, 1.0), (1.0, 0.5), (2.0, 1.0)]:
            H_mpo = MPO.from_bonds(N, 2, dtype=DTYPE).generate_xxz(Jz=Jz, Jxy=Jxy).to_dense()
            H_ref = xxz_dense(N, Jz=Jz, Jxy=Jxy, dtype=DTYPE)
            assert tc.allclose(H_mpo, H_ref, atol=1e-12)

def test_xxz_mpo_hermitian():
    for N in [2, 4, 6]:
        H = MPO.from_bonds(N, 2, dtype=DTYPE).generate_xxz().to_dense()
        assert tc.allclose(H, H.conj().T, atol=1e-12)

def test_xxz_mpo_open_boundary_shape():
    mpo = MPO.from_bonds(6, 2, dtype=DTYPE).generate_xxz()
    assert mpo.tensors[0].shape[0] == 1 and mpo.tensors[5].shape[1] == 1   # 开端
```

> 注意：测试文件从 `tests/reference_models.py` 导入 dense 参考（它再导出
> `latticetn.operators`），不直接从 `latticetn.operators` 导。把你的
> `<model>_dense` 再导出也加到那里。

### 4. native 能量测试 — `tests/test_<model>_native_energy.py`

断言可微 native 收缩 `rayleigh_energy_native(mps, mpo)` 与 dense 能量在已知
态上一致。模板：`tests/test_native_mpo_energy_contraction.py`。

```python
def test_xxz_native_energy_matches_dense():
    N = 4
    mpo = MPO.from_bonds(N, 2, dtype=DTYPE).generate_xxz()
    mps = MPS(N, 2, 4, dtype=DTYPE)
    e_native = rayleigh_energy_native(mps, mpo)
    e_dense  = (mps.to_dense().conj() @ mpo.to_dense() @ mps.to_dense()).real / (mps.to_dense().conj() @ mps.to_dense()).real
    assert abs(float(e_native) - float(e_dense)) < 1e-8
```

### 5. 观测量（可选）

若模型有自然观测量（如 staggered 磁化），加 dense + native 变体并交叉验证，
仿 `latticetn.observables`。

### 6. benchmark 脚本 — `scripts/run_<model>_benchmark.py` + `scripts/<model>_score.py`

仿 `scripts/run_ad_mps_heisenberg.py` / `ad_two_site_score.py`：构建新 MPO，
用 `train_ad_*` 训练，对比精确（小 N）与 DMRG，写报告。保留 score 脚本的
required-terms 检查。

### 7. AD 求解器

用**既有**求解器在新 MPO 上训练——不新增算法：

```python
mps = MPS(N, 2, chi, dtype=tc.complex128)
mpo = MPO.from_bonds(N, 2, dtype=tc.complex128, device="cpu").generate_xxz(Jz=1.0, Jxy=0.5)
res = train_ad_two_site(mps, mpo, num_sweeps=4, local_steps=20, lr=1.0,
                        optimizer="lbfgs", max_bond_dim=chi)
```

### 8. 基线对比

把 AD 最终能量与精确（小 N）和经典 DMRG（`dmrg.run_dmrg`）对比。AD 能量
必须**在/高于**精确（容差内），且**绝不低于基态**超过 `1e-8`。DMRG 是
**参考基线**，绝非 AD 主线。

## 运行命令

```bash
# MPO-to-dense + native 能量测试：
python -m pytest -q tests/test_xxz_mpo_dense.py tests/test_xxz_native_energy.py

# 新模型完整 score（score 脚本就绪后）：
python scripts/xxz_score.py --fast
```

## 预期输出

- `test_xxz_mpo_matches_dense_several_sizes` → PASS（atol=1e-12）。
- AD 求解器在模型容差内达到精确基态能量（如 N=6 良好条件数下 `1e-5`），
  `below_ground=False`。

## 常见错误

- **MPO-to-dense 偏差 > 1e-12** —— MPO 构造器的 start/pair 算符或边界处理
  有误。对照 Heisenberg 构造器的 5 态模式；检查左边界只留 row 0、右边界只留
  末列。
- **能量差 4 倍** —— 用了 Pauli `σ` 而非 `S = σ/2`。dense 参考用
  `spin_operators(...)`，不要用 `pauli_matrices()`。
- **`below_ground = True`** —— bug。查厄米性（`H == H.conj().T`）、dtype
  （`complex128`）、MPO 是否意外非厄米。
- **在能量路径里放 `eigh`/`svd`/`qr`** —— 别。loss 路径须保持 autograd-clean；
  AST policy 测试会强制。
- **从 AD 模块 import `dmrg`/`lanczos`** —— 禁止；它们只是参考基线，从 score
  脚本用，绝不在 loss 路径。

## 要保持的 convention

- `S = σ/2`（绝不与 Pauli 混用）。
- 开边界；MPS `(left, phys, right)`，MPO `(left, right, phys_in, phys_out)`。
- 默认 `torch.complex128`，默认 device CPU。
- loss 路径保持 autograd-clean（见 `docs/AD_MAINLINE_POLICY.md`）。
